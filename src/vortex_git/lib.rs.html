1//! # Vortex Git Backend
2//!
3//! Git-based configuration source for Vortex Config Server.
4//!
5//! This crate provides a Git backend implementation that can clone repositories,
6//! read configuration files, and support Spring Cloud Config file conventions.
7//!
8//! ## Features
9//!
10//! - Git operations via system `git` CLI for maximum compatibility
11//! - Async trait-based configuration source abstraction
12//! - Support for branches, tags, and commit references
13//! - Background refresh with configurable intervals
14//! - Spring Cloud Config compatible file resolution
15//!
16//! ## Example
17//!
18//! ```ignore
19//! use vortex_git::{GitBackend, GitBackendConfig, ConfigSource, ConfigQuery};
20//!
21//! let config = GitBackendConfig::builder()
22//!     .uri("https://github.com/org/config-repo.git")
23//!     .local_path("/tmp/config-repo")
24//!     .default_label("main")
25//!     .build()?;
26//!
27//! let backend = GitBackend::new(config).await?;
28//!
29//! let query = ConfigQuery::new("myapp", vec!["dev"]);
30//! let result = backend.fetch(&query).await?;
31//! ```
